## Task 1 — Strip GM director instruction from first stream message (LR-006)

**File:** `server/gmOrchestrator.ts`
**Function:** Inside `runGM` — the inner `extractAndStreamNarrative` closure

**Problem:** The GM's first streaming message echoes back the director instruction block,
e.g. `*Campaign start. Drop the player straight into the middle...*`. The existing filter
strips JSON blocks but not this leading `*...*` director block.

**Change — Step 1:** Add two new flags alongside the existing `inJsonBlock` flag.
Find the line `let inJsonBlock = false;` inside `runGM` and add the two lines immediately after:

```typescript
let inJsonBlock = false;
let directorBlockSkipped = false;   // ADD
let inDirectorBlock = false;         // ADD
```

**Change — Step 2:** At the very top of `extractAndStreamNarrative`, before the existing
`if (inJsonBlock) return;` line, add the following block:

```typescript
// LR-006 FIX: Strip leading *UPPERCASE...* director instruction block
if (!directorBlockSkipped) {
  if (!inDirectorBlock) {
    const trimmed = fullText.trimStart();
    if (/^\*[A-Z]/.test(trimmed)) {
      inDirectorBlock = true;
    } else if (fullText.length > 5) {
      // No director block present — proceed normally
      directorBlockSkipped = true;
    }
  }
  if (inDirectorBlock) {
    // Find the end of the director block: closing * or double newline
    const openIdx = fullText.indexOf('*');
    const closeAsterisk = fullText.indexOf('*', openIdx + 1);
    const doubleNewline = fullText.indexOf('\n\n');
    const closingIdx = [closeAsterisk, doubleNewline]
      .filter(i => i > openIdx)
      .sort((a, b) => a - b)[0] ?? -1;
    if (closingIdx > 0) {
      // Director block complete — skip it, mark position, continue streaming
      streamedNarrative = fullText.slice(0, closingIdx + 1);
      directorBlockSkipped = true;
      inDirectorBlock = false;
      // Fall through to normal streaming with remaining content
    } else {
      return; // Block not yet complete, wait for more chunks
    }
  }
}
```

**Verify:** Start a new campaign → Begin Adventure → confirm the first GM message is
narrative prose only with no `*Campaign start...*` or `*...*` prefix visible to the player.

## Task 4 — World-consistency guardrail in GM system prompt (UXF-007)

**File:** `server/gmOrchestrator.ts`
**Function:** `buildSystemPrompt`

**Problem:** The GM narrates any player action regardless of whether it's physically
possible in the world (e.g., a medieval fighter "turning into a spaceship"). There is no
genre guardrail.

**Change:** Find the `buildSystemPrompt` function. Inside the returned system prompt
template string, locate the section that describes campaign constraints or player rules
(near content about dice rolls, time skips, or the character/party block). After that
section, add the following guardrail text directly into the prompt string:

```
WORLD INTEGRITY — GENRE GUARDRAIL:
You are the Game Master of a [insert campaign setting here — use campaign.setting or equivalent variable]. This world has a consistent internal reality with its own physics, technology level, and rules for what is possible.

When a player describes an action that violates this world's genre and physics (e.g. a medieval fighter becoming a spaceship, a contemporary character summoning magic that does not exist in this world):
1. Acknowledge the impulse briefly — one sentence, in-character, with personality
2. Redirect to a plausible in-world alternative that captures the spirit of the action
3. Keep the pace moving — do not lecture, do not break immersion

Examples:
- Medieval fighter tries to "turn into a spaceship" → Brief wry acknowledgment + offer a plausible alternative (flee on horseback, climb a wall, use a grapple)
- Space captain tries to "cast a fireball" → Redirect to tech equivalent (plasma grenade, thruster burst) unless magic is established in this campaign's lore

EXCEPTION: If the player's action is consistent with this specific campaign's established lore (e.g. a world where technology and magic coexist), lean into it. Only redirect actions that violate THIS world's own rules.

Keep redirects to 1–2 sentences. In-world voice only. Then advance the scene.
```

Use the correct template literal syntax to embed `campaign.setting` (or whatever the
variable is called) into the guardrail text.

**Verify:** In a medieval campaign, type: "I grow dragon wings, turn into a spaceship, and
fly to the moon." The GM should respond with a brief in-character redirect (not narrate the
transformation).
