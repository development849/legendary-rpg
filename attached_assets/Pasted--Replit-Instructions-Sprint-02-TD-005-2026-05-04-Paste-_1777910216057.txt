## Task 1 — GM streaming error boundary (TD-005)

**File:** `server/gmOrchestrator.ts`  
**Function:** `runGM`

**Problem:** If the GPT-4o streaming call fails (timeout, rate limit, API error) and the
silent non-streaming fallback also fails, `onDone` is never called. The route handler's
`onDone` is the only place that calls `res.end()` — so the SSE connection hangs open
indefinitely. The player sees a frozen, incomplete chat bubble with no error message and no
way to recover without a hard refresh. Additionally, the current catch block swallows the
original error entirely (no logging).

**Find this exact block in `runGM`:**

```typescript
  } catch (err: any) {
    const response = await openai.chat.completions.create({
      model: "gpt-4o",
      messages,
      stream: false,
      max_tokens: 2000,
      temperature: 0.8,
    });
    fullText = response.choices[0]?.message?.content ?? "";
  }
```

**Replace it with:**

```typescript
  } catch (err: any) {
    // TD-005 FIX: log the primary streaming failure before attempting fallback
    console.error("[GM] Streaming call failed, attempting non-streaming fallback:", err?.message ?? err);

    try {
      const response = await openai.chat.completions.create({
        model: "gpt-4o",
        messages,
        stream: false,
        max_tokens: 2000,
        temperature: 0.8,
      });
      fullText = response.choices[0]?.message?.content ?? "";
    } catch (fallbackErr: any) {
      // TD-005 FIX: fallback also failed — surface a friendly error and close the SSE connection cleanly
      console.error("[GM] Non-streaming fallback also failed:", fallbackErr?.message ?? fallbackErr);
      onChunk("\n\n*The Game Master lost the thread. Please try your action again.*");
      onDone("", [], [], [], undefined, [], undefined);
      return;
    }
  }
```

**Where to find it:** Search for `const response = await openai.chat.completions.create` —
there will be exactly one match inside a `catch` block. That is the target. The surrounding
code (the `extractAndStreamNarrative` function above it and the JSON parse block below)
stays exactly as-is.
