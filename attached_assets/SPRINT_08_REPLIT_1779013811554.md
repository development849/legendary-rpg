## Step 2 — Add API endpoints for saving and retrieving feedback

**Paste into Replit AI:**

```
In `server/routes.ts`, add two new API routes:

**Route 1 — POST /api/feedback**
- Requires authenticated user (same auth middleware as other protected routes)
- Body: `{ sessionId, rating, comment }` where rating is 1–5 integer, comment is optional string ≤200 chars
- Validate: rating must be between 1 and 5 inclusive; if comment is provided, trim it and cap at 200 chars
- Insert into `session_feedback` table with: sessionId, userId (from req.user.id or equivalent), rating, comment, createdAt = now()
- Return 201 on success: `{ success: true }`
- Return 400 if rating is missing or out of range

**Route 2 — GET /api/admin/feedback**
- Protected by a simple secret check: require header `x-admin-key: LEGENDARY_ADMIN` (or read the value from an env var called `ADMIN_KEY` if one already exists; if not, use the hardcoded string as a fallback)
- Returns all rows from `session_feedback`, ordered by createdAt DESC, joined with a user email if easily available (optional)
- Return as JSON array

No frontend changes yet — just the two routes.
```

**✓ Success check:** `POST /api/feedback` with `{"sessionId": 1, "rating": 4, "comment": "Fun!"}` returns 201. `GET /api/admin/feedback` with header `x-admin-key: LEGENDARY_ADMIN` returns a JSON array (may be empty).

**Paste into Replit AI:**

```
Create a new file `client/src/components/SessionFeedback.tsx`.

This is a modal overlay that appears when a game session ends. It should:

   - Heading: "How was your adventure?"
```
