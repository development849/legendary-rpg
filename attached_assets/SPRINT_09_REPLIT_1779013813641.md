## Overview

This makes Legendary RPG usable on mobile phones (390px viewport — iPhone 14 baseline). Many beta players will arrive on phones from Reddit/Product Hunt links. The approach is responsive CSS — no new pages or routes. One breakpoint: `max-width: 768px`. No mobile-specific build.

**Four flows, in priority order:**

| Step | Flow | Complexity | Estimated time |
|---|---|---|---|
| 1 | Game session | High | 20–25 min |
| 2 | Character creation | Medium | 15–20 min |
| 3 | Dashboard | Low | 10 min |
| 4 | Auth page | Low | 10 min |

**Total estimated Replit time:** 55–65 minutes across all steps.

## Step 1 — Game Session Mobile Layout

**Paste into Replit AI:**

```
Make the game session screen (`client/src/pages/game-session.tsx` and any related component files such as `CharacterSheet`, `QuickActions`, `ChatInput`, or `GameHeader`) responsive at mobile widths (max-width: 768px).

Apply the following changes using CSS (Tailwind classes, inline styles, or a CSS module — whichever pattern the project already uses consistently):

**1a. Session header (top bar):**
- On mobile: reduce height to 48px minimum
- Show icon-only buttons for "Character" (opens character sheet) and "Leave Session" (triggers exit flow) — remove text labels from header buttons at mobile widths
- Each icon button must have a touch target of at least 44×44px (add padding if needed) and an aria-label attribute
- Campaign name: truncate at ~20 characters with ellipsis (`overflow: hidden; text-overflow: ellipsis; white-space: nowrap; max-width: calc(100% - 96px)`)

**1b. Chronicle scroll (main content area):**
- Full width at mobile, no fixed left percentage
- GM message bubbles: align left, max-width 85% of viewport
- Player message bubbles: align right, max-width 85% of viewport
- Font size on bubble text: 15px at mobile (from 16px on desktop)
- Padding per bubble: 12px horizontal, 8px vertical

**1c. Quick action pills:**
- On mobile: render as a horizontal scroll row above the input bar
- `display: flex; flex-direction: row; overflow-x: auto; -webkit-overflow-scrolling: touch`
- Hide the scrollbar: `scrollbar-width: none` (Firefox) and `::-webkit-scrollbar { display: none }` (WebKit)
- Each pill: 36px height, min-width 80px, `white-space: nowrap`
- Add a subtle fade-out gradient on the right edge to hint at scrollability (use a `::after` pseudo-element with `background: linear-gradient(to right, transparent, #0d0d12)`)

**1d. Chat input bar (bottom):**
- On mobile: `position: fixed; bottom: 0; left: 0; right: 0`
- Minimum height: 56px
- Add `padding-bottom: env(safe-area-inset-bottom)` to handle iPhone home bar
- Send button: 44×44px minimum, amber background (#f59e0b)
- Text input grows up to 3 lines, then scrolls internally (`resize: none; overflow-y: auto; max-height: 80px`)

**1e. Character sheet panel (off-canvas drawer):**
- On mobile, the character sheet sidebar should be hidden by default
- It opens as a drawer sliding in from the right when the "Character" icon button in the header is tapped
- Drawer CSS: `position: fixed; top: 0; right: 0; height: 100%; width: 90vw; max-width: 360px; transform: translateX(100%); transition: transform 0.25s ease; z-index: 50; background: #13131a; overflow-y: auto`
- When open: `transform: translateX(0)`
- Behind the drawer: a dark overlay `position: fixed; inset: 0; background: rgba(0,0,0,0.6); z-index: 40` — tapping the overlay closes the drawer
- Add React state to track whether the drawer is open (`const [sheetOpen, setSheetOpen] = useState(false)`)
- Stats grid inside the drawer: use 2-column layout at mobile (`display: grid; grid-template-columns: 1fr 1fr; gap: 8px`)
- Close: tap overlay or a close button inside the drawer

**1f. Character hint pill (FR-002):**
- Ensure `bottom` value is at least `120px` on mobile to clear the fixed input bar and pills row
- The current value was `bottom: 24px` — change to `bottom: 120px` at `max-width: 768px`

**1g. Keyboard handling (iOS):**
- When the soft keyboard appears on iOS, the fixed input bar should remain visible above the keyboard
- Add a `visualViewport` resize listener that adjusts the input bar's bottom offset:
  ```
  useEffect(() => {
    const viewport = window.visualViewport;
    if (!viewport) return;
    const handler = () => {
      const offset = window.innerHeight - viewport.height;
      // Apply offset to the input bar's bottom position
    };
    viewport.addEventListener('resize', handler);
    return () => viewport.removeEventListener('resize', handler);
  }, []);
  ```
- After the keyboard dismisses, restore the default `bottom: env(safe-area-inset-bottom)`

After applying all changes above, do a full check: at 390px viewport width, scroll through a sample session. Confirm no horizontal overflow by running in the browser console: `document.documentElement.scrollWidth > window.innerWidth` — this must return false.
```

**✓ Success check:** At 390px (Chrome DevTools device emulation — iPhone 14 portrait):
- Chronicle scroll fills the viewport; no horizontal scroll
- Quick action pills scroll horizontally without wrapping
- Chat input is fixed at the bottom, above the home indicator area
- Tapping "Character" icon opens the drawer from the right; tapping the overlay closes it
- Character hint pill (if present) does not overlap the input bar
- `document.documentElement.scrollWidth > window.innerWidth` returns `false`

## Step 2 — Character Creation Mobile Layout

**Paste into Replit AI:**

```
Make the character creation flow (`client/src/pages/character-creation.tsx` or equivalent multi-step form) responsive at mobile widths (max-width: 768px).

Apply the following changes:

**2a. Step indicator:**
- Desktop: numbered step tabs across the top
- Mobile: replace with a progress bar
  - `height: 4px; background: #1e1e2a; border-radius: 2px` as the track
  - Filled portion: `background: #f59e0b; border-radius: 2px` — width is `(currentStep / totalSteps * 100)%`
  - Below the bar: centred step label in Inter, 13px, muted colour: e.g. "Step 2 of 5: Choose Class"
  - Hide the original tab-style step indicator on mobile

**2b. Class and race selection cards:**
- Desktop: typically 3–4 column grid
- Mobile: 2-column grid (`display: grid; grid-template-columns: 1fr 1fr; gap: 12px`)
- Each card: full width within its column; image/icon at top; name + brief description below; minimum tap target height: 80px
- The selected state (ring/border) must still be clearly visible on mobile card size

**2c. Form inputs:**
- All text inputs, textareas, and selects: full width (`width: 100%`) on mobile
- Character name field: full width; if the FR-007 randomise dice button exists, position it to the right inside the input container (44×44px icon button)
- Backstory textarea: full width, minimum height 120px
- Remove any `max-width` container constraints at mobile widths — allow full 390px width with 16px padding on each side

**2d. Portrait generation step:**
- Portrait preview image: full width, max-height 300px, `object-fit: cover`
- "Generate Portrait" button: full width, amber fill, 48px height
- Any "Regenerate" or secondary button: full width below the primary

**2e. Sticky CTA bar (bottom navigation — fixes UXF-003):**
- On mobile, add a fixed bottom bar containing the "Next" (or "Continue") button:
  ```
  position: fixed;
  bottom: 0;
  left: 0;
  right: 0;
  padding: 12px 16px;
  padding-bottom: calc(12px + env(safe-area-inset-bottom));
  background: #13131a;
  border-top: 1px solid #1e1e2a;
  ```
- "Next" button inside bar: full width, amber fill, 48px height
- "Back" link: small text link, centred above the bar or to the left side of the bar if horizontal space allows
- Add `padding-bottom: 80px` to the main scrollable content so it's not obscured by the fixed bar

After applying, verify at 390px: scroll through each step of character creation. The "Next" button must always be visible without scrolling.
```

**✓ Success check:** At 390px:
- Progress bar visible at top of each step; no tab-style step indicator
- Class/race cards appear in a 2-column grid; tap targets ≥80px height
- All form inputs are full width; no horizontal scroll
- "Next" button is fixed at the bottom of every step, always visible
- `document.documentElement.scrollWidth > window.innerWidth` returns `false` on every step

## Step 3 — Dashboard Mobile Layout

**Paste into Replit AI:**

```
Make the dashboard page (`client/src/pages/dashboard.tsx`) responsive at mobile widths (max-width: 768px).

Apply the following changes:

**3a. Navigation header:**
- On mobile: 56px height

**3d. Sidebar (stats, recent activity, or any right-panel content):**
- Hide entirely on mobile (`display: none` at max-width: 768px)
- If any critical stat (e.g. total sessions played) lives only in the sidebar, move it as a metadata line inside each campaign card
- Do not collapse to accordion — simply hide

After applying, verify at 390px: dashboard loads showing 1-column campaign cards and stacked CTAs with no horizontal scroll.
```

**✓ Success check:** At 390px:
- Campaign cards stack in a single column; no 2-column grid
- "New Chronicle" and "Create Hero" CTAs are full width and vertically stacked
- Any sidebar content is not visible
- `document.documentElement.scrollWidth > window.innerWidth` returns `false`
