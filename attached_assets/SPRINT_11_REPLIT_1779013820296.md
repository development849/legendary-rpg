## Step 2 — Add the sticky bottom CTA bar (UXF-003 + UXF-004 combined)

**In the Replit AI chat, paste:**

```
In the character creation wizard (the file we just confirmed), I need to add a sticky bottom bar that keeps the Next button always visible regardless of scroll position. The bar should also correctly disable the Next button until the player has made a valid selection for the current step.

Apply the following changes:

1. Add a sticky bottom bar component at the bottom of each wizard step's JSX (or add it once as a shared wrapper if the wizard has a shared layout). The bar should be:
   - position: fixed, bottom: 0, left: 0, right: 0
   - height 72px with padding 0 24px
   - background colour matching the app's dark theme (approximately #1a1b26 or whatever the main background is)
   - a top border using a subtle rgba line (rgba(255,255,255,0.08) or similar)
   - z-index: 50 so it sits above content
   - flex layout: step indicator on the left, Next button on the right

2. The Next button inside the bar should:
   - Be DISABLED and visually muted (opacity 0.5, cursor not-allowed) when the current step has no valid selection
   - Be ENABLED and styled with the app's primary gold/amber colour when a valid selection exists
   - Have a wrapper div with a title attribute (tooltip) that shows the reason when disabled (e.g. "Choose a class to continue" on the class step, "Choose a race to continue" on the race step, "Generate your portrait to continue" on the portrait step)

3. Add paddingBottom: 88px (or equivalent) to the main page content container so content doesn't hide behind the sticky bar on scroll.

4. The valid-selection condition per step:
   - Class step: selectedClass is truthy
   - Race step: selectedRace is truthy
   - Portrait step: character has a profilePicture URL (non-empty string)
   - Name/backstory steps: name.trim().length > 0 and all required fields filled

If there is an existing Next button in the page body, either remove it (the sticky bar replaces it) or keep it but change it to a secondary style so it doesn't compete with the sticky bar.

After applying: confirm the app loads (HTTP 200) and tell me which files were changed.
```
